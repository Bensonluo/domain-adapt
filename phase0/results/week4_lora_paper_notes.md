# LoRA 论文要点

## 1. 核心公式

```
W_new = W_0 + ΔW
      = W_0 + (α/r) × B @ A
```

- `W_0`: 原始冻结权重，shape = (d_out, d_in)
- `A`: 下投影，shape = (r, d_in)，随机高斯初始化
- `B`: 上投影，shape = (d_out, r)，**初始化为 0**
- `r`: 秩，通常 4-64
- `α`: 缩放因子，`scaling = α / r`

**为什么 B=0？** 训练开始时 ΔW = B @ A = 0 @ A = 0，模型输出和原始预训练模型完全一样。如果 B 也随机初始化，训练第一步就会破坏预训练权重。

## 2. SVD 视角（为什么有效）

```
ΔW = B @ A    shape: (d_out, d_in), 但秩只有 r

等价于: 预训练权重的变化矩阵 ΔW 是低秩的
即: 模型适应新任务时，权重的改变不需要完整的 d_out × d_in 个自由度
r = 8 就够了 → 参数从 d_out × d_in 降到 r × (d_out + d_in)
```

论文的实证：对 GPT-3 175B 微调后的 ΔW 做 SVD，发现前 20 个奇异值就包含了大部分信息。权重变化本身就是低秩的，所以用低秩近似不会损失多少信息。

## 3. 参数量对比

```
全量微调: 每层 d_out × d_in 个参数
LoRA:     每层 r × (d_out + d_in) 个参数

例: d_in = d_out = 4096, r = 8
  全量: 4096 × 4096 = 16,777,216
  LoRA: 8 × (4096 + 4096) = 65,536
  比例: 65,536 / 16,777,216 ≈ 0.4%
```

## 4. 为什么只适配 q_proj 和 v_proj（Table 4）

论文实验（GPT-3 175B, WikiSQL, MultiNLI）：

| 适配模块 | WikiSQL | MultiNLI | 可训练参数 |
|---------|---------|----------|-----------|
| 只适配 q | 73.2 | 89.6 | 基准 |
| 只适配 k | 72.9 | 89.3 | 同上 |
| 只适配 v | 73.1 | 89.6 | 同上 |
| **q + v** | **73.7** | **90.0** | **2x** |
| q + k + v + o | 73.9 | 90.2 | 4x |
| 全部（含 FFN）| 74.0 | 90.3 | 很多 |

结论：q+v 只用 2x 参数就拿到了接近全部适配的 95%+ 效果。性价比最高。

**为什么不加更多？** 从 q+v 到全部适配，效果只提升 0.3%，但参数翻很多倍。实际中如果你训得动，加更多模块（k, o, FFN 的 gate/up/down）通常更好。

## 5. alpha 参数的含义

```
scaling = alpha / rank
ΔW 实际贡献 = (alpha / r) × B @ A
```

- `alpha` 是一个固定常数（常见值 16 或 32）
- `scaling = alpha / r`，当你增大 rank 时，每个 LoRA 分量的贡献自动缩小
- 好处：**换 rank 不用重新调学习率**。alpha 不变，scaling 随 rank 自动调整
- 工程上的惯例：`alpha = 2 × rank`，这样 `scaling = 2`，和原始权重的量级匹配

## 6. 推理无额外开销

训练结束后把 LoRA 权重 merge 回原始权重：

```python
W_merged = W_0 + (alpha / r) * B @ A
```

merge 后 LoRA 适配器可以丢掉，推理时就是普通的矩阵乘，零额外开销。这比 adapter（多一层网络）和 prefix-tuning（多一段 prefix 占 KV cache）都好。

## 7. Rank 怎么选

| 场景 | 推荐 rank | 原因 |
|------|----------|------|
| 简单任务（单领域 SFT） | 4-8 | 权重变化简单，低秩就够 |
| 复杂任务（多领域/推理） | 16-32 | 需要更多自由度 |
| 通用微调 | 8-16 | 平衡点 |
| 大模型 (>7B) + 简单任务 | 4 | 模型容量大，适应空间小 |

rank 太小（1-2）：欠拟合，学不到任务细节
rank 太大（64-128）：接近全量微调，失去参数效率优势，容易过拟合
