# Self-Instruct 论文精读笔记

> 论文: SELF-INSTRUCT: Aligning Language Models with Self-Generated Instructions (Wang et al., ACL 2023)
> 链接: https://arxiv.org/abs/2212.10560
> 基座: GPT-3 (davinci, 175B)
> PDF: [../week18/papers/Self-Instruct_2212.10560.pdf](../week18/papers/Self-Instruct_2212.10560.pdf)

---

## 核心观点 (TL;DR)

```
指令微调（instruction tuning）依赖人工写的指令数据，贵、少、不够多样。
SELF-INSTRUCT 的核心：让 LLM 自己给自己造指令数据。

流程（自举 bootstrap）:
  175 条人工种子指令
     → LLM 生成新指令
     → 过滤（去重/去噪）
     → LLM 生成对应 input-output 实例
     → 再过滤
     → 加回指令池
     → 循环……
  最终 ~52K 指令 / ~82K 实例，全靠模型自己造，人工只写 175 条种子。

效果: 用这批数据微调 GPT-3，SUPERNI 上 +33%，逼近 InstructGPT001。
      而且是"几乎零标注成本"的对齐方法。

一句话：合成数据生成的范式起点。把"人类标注"这个瓶颈，换成"模型自举"。
      这是 response distillation 的数据层基础——没有它，就没有后面的
      Alpaca / Vicuna / Zephyr 整条蒸馏链。
```

---

## 核心问题

instruction-tuned 模型（InstructGPT、FLAN、T0）很强，但它们依赖**人工写的指令数据**（SUPERNI、PROMPTSOURCE）。这种数据有两个瓶颈：

1. **成本高**：要专家写任务、写答案，标注几万条要花大量人力。
2. **多样性差**：人倾向于写"熟悉的 NLP 任务"（分类、QA、翻译），覆盖不到真实用户的五花八门需求。

能不能让模型自己造数据，绕开人工标注？——这就是 Self-Instruct 要回答的。

## 方法：四步自举循环

```
┌─────────────────────────────────────────────────────────────┐
│  种子池: 175 条人工指令 (25 分类 + 150 非分类)                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
  Step 1: 指令生成 (Instruction Generation)
    从池子里采 8 条做 in-context 示例 (6 人工 + 2 模型生成的)
    → 让 LM 续写 "Task 9: ..." "Task 10: ..."
    → 得到一批新指令
                              │
                              ▼
  Step 2: 任务类型识别 (Classification ID)
    用 few-shot prompt 判断每条新指令是不是分类任务
    (分类任务 vs 非分类任务，用不同的实例生成策略)
                              │
                              ▼
  Step 3: 实例生成 (Instance Generation)
    分类任务 → Output-first: 先生成类标，再按每个类标生成输入
              (避免输入偏向某个类，如语法错误检测全生成正确句子)
    非分类任务 → Input-first: 先生成输入，再生成输出
                              │
                              ▼
  Step 4: 过滤 (Filtering)
    - ROUGE-L 相似度过滤: 新指令和池里任何一条 > 0.7 就丢
      (防止指令池被重复项淹没，保证多样性)
    - 关键词过滤: 含 image/picture/graph 等图片类词的丢
      (LM 处理不了多模态)
    - 启发式过滤: 太长/太短/输出重复输入的丢
                              │
                              ▼
  合格的指令+实例 → 加回池子 → 回到 Step 1 继续自举
                              │
                              ▼
  最终: 52,445 指令 + 82,439 实例
        → 标准 SFT 微调原 LM (GPT-3)
```

**关键设计点**：
- **8 条示例里混 2 条模型生成的**：促进多样性，不让生成分布被人工种子锁死。
- **分类任务单独走 Output-first**：这是论文的一个实用创新，解决"输入偏向多数类"的问题。
- **ROUGE-L 0.7 阈值**：控制多样性，避免生成器陷入"换个说法重复同一任务"。

## 数据质量：能信吗？

人工抽检 200 条（Table 2）：

| 检查项 | Yes % |
|--------|-------|
| 指令描述了一个有效任务？ | **92%** |
| 输入和指令匹配？ | 79% |
| 输出是正确可接受的回答？ | 58% |
| 三个字段全部有效 | 54% |

```
解读:
  - 指令质量高 (92%) —— 模型很会"出题"
  - 输出质量一般 (58%) —— 模型"答题"有错，含噪声
  - 但关键是: 即使有噪声，这批数据微调后仍大幅涨点
            (SFT 对噪声有一定鲁棒性，且数据量/多样性弥补了单条质量)

这是合成数据的核心 trade-off:
  单条质量 < 人工标注，但 数量 × 多样性 远超人工 → 净赚
```

## 关键实验与数字

### Experiment 1: SUPERNI 零样本泛化（Table 3）

| 模型 | 参数量 | ROUGE-L |
|------|-------|---------|
| T5-LM (vanilla) | 11B | 25.7 |
| GPT-3 (vanilla) | 175B | 6.8 |
| T0 | 11B | 33.1 |
| GPT-3 + T0 训练 | 175B | 37.9 |
| **GPT-3 + SELF-INSTRUCT** | **175B** | **39.9** |
| InstructGPT001 | 175B | 40.8 |

- vanilla GPT-3 几乎不会跟指令（6.8），SELF-INSTRUCT 微调后 **+33.1 个点**到 39.9
- **逼近 InstructGPT001**（40.8），而 InstructGPT 用了私有用户数据 + 人工标注
- 超过用人工 T0 数据微调的版本（37.9）——证明合成数据 ≥ 部分人工数据

### Experiment 2: 252 条真实用户指令（人工评估，Figure 6）

- 4 级评分（A 满意 / B 小瑕疵可接受 / C 有错 / D 无关）
- GPT-3 SELF-INST 把"可接受以上"(A+B) 当合格，只比 InstructGPT001 **落后 5%**
- 大幅超过 T0 / SUPERNI 训练的版本
- InstructGPT002/003 仍领先（它们用了更多数据和 RLHF）

### 数据规模 & 质量的作用（Figure 7）

```
数据量曲线:
  175 条 (只用种子) → 31.0%  A 级
  800 条            → 36.9%
  6,400 条          → 43.7%
  16,000 条         → ~44%   ← 开始平台期
  51,200 条         → 44.4%  ← 边际收益很小了

平台期出现在 ~16K: 说明数量不是越多越好，多样性比数量重要。
                   这也是为什么他们停在 52K。

质量提升实验:
  用 InstructGPT003 重新生成所有 output (蒸馏更强模型)
  → 同样数据量，A 级率从 44.4% → 54.4% (+10%)

  启示: 合成数据 = 「量」 × 「质」。
        Self-Instruct 解决了"量"，质量可以靠"再蒸馏强模型"提升。
        → 这正是 Zephyr 路线的思路（UltraChat 用 GPT-3.5 造，UltraFeedback 用 GPT-4 打分）
```

## 为什么有效？（合成数据的底层逻辑）

```
为什么模型自己造的数据，能教会模型自己?

1. 预训练的 LM 已经"知道"很多任务，只是不知道"什么时候该用什么"
   (latent capability，能力是潜在的)
   → 指令微调不是教新能力，是"激活+对齐"已有能力

2. 自举生成 = 把模型的 latent capability 显式化成 (指令, 输出) 对
   → 等于让模型把自己"会的事"写下来，再回头学
   → 论文原话: "a model's knowledge is distilled to itself"

3. 多样性 > 数量
   种子越多样 → 生成越多样 → 覆盖的任务分布越广 → 泛化越好
   (Figure 3 的 verb-noun 分析显示生成了非常多样的意图)

4. 对噪声鲁棒
   SFT 用交叉熵，单条错误数据影响有限;
   数据多了，正确的信号 dominate。
   所以 58% 正确率的输出也能训出好模型。

本质上: Self-Instruct 是一种"自我蒸馏"——
        teacher 和 student 是同一个模型，
        蒸馏的内容是"指令任务的形式"而非"知识本身"。
```

## 局限性（论文自己承认的）

```
1. 长尾任务覆盖差
   LM 对高频任务造得好，冷门/创意任务造不好
   (继承 LM 的长尾弱点)

2. 依赖大模型
   自举要 in-context learning，小模型做不了
   → 这条技术线天然有"大模型门槛"

3. 放大偏见
   迭代生成会放大 LM 已有的社会偏见
   (模型觉得什么任务"正常"，就更多地生成什么)

4. 输出质量天花板 = teacher 模型
   GPT-3 造的数据，质量上限就是 GPT-3 的水平
   → 所以后续工作改用更强的模型造数据 (Alpaca 用 text-davinci-003)
```

## 我的 takeaway

1. **Self-Instruct 是合成数据的开山之作**。它把"指令微调"的瓶颈从"人工标注"换成了"模型自举"，直接催生了 Alpaca、Vicuna、WizardLM 等一整代开源模型。
2. **核心洞察：能力是潜在的，指令微调是"激活"而非"注入"**。模型预训练时已经见过海量任务模式，只是没被"指令格式"对齐。Self-Instruct 用模型自己把这些模式显式化。
3. **多样性 > 质量 > 数量**。ROUGE-L 过滤 + 平台期出现在 16K，都说明"覆盖广"比"每条精"重要。这是合成数据区别于人工数据的关键。
4. **它是一种"自我蒸馏"**——teacher=student=同一个模型，蒸馏的是"任务形式"。这种"自举"思想后来在 R1 的 rejection sampling（用 RL 探索到的好推理固化进 SFT 数据）里再次出现，是蒸馏 + RL 的交汇点。
5. **和我的方向（垂域大模型）直接相关**：垂域里"领域指令数据"稀缺，Self-Instruct 提供了"用领域种子 + 大模型自举"造领域指令数据的模板。week10 的数据准备可以直接借鉴这套 pipeline。
