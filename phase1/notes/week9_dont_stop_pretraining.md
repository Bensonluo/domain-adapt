# Don't Stop Pretraining 论文精读笔记

> 论文: Don't Stop Pretraining: Adapting Language Models to Domains and Tasks (Gururangan et al., ACL 2020)
> 链接: https://arxiv.org/abs/2004.10964
> 基座: RoBERTa

---

## 核心观点 (TL;DR)

```
在下游任务微调前，先在领域/任务数据上继续做 MLM 预训练（不加任何 label），
几乎总是能涨点 —— 即使基座 RoBERTa 已经很强。

两个变体：
  DAPT = 拿整个「领域」语料继续练   (贵，但收益最大)
  TAPT = 拿「这一个任务」的数据继续练 (便宜，几乎免费涨点)
  DAPT + TAPT 叠加 = 最强

一句话：预训练不是终点，"接着练"才是领域自适应的起点。这是 CPT 这条线的开山之作。
```

---

## 核心问题

通用预训练模型（RoBERTa）已经见过海量文本，为什么在医疗、法律、金融这种专业任务上还是不够好？再拿领域数据继续 MLM 一遍，到底值不值？

---

## 两个变体：DAPT vs TAPT

```
                       预训练基座 (RoBERTa, 海量通用语料)
                                  │
                 ┌────────────────┼────────────────┐
                 ▼                                  ▼
           DAPT 路线                           TAPT 路线
  「整个领域」继续 MLM                「这一个任务的训练集」继续 MLM
   例：所有生物医学论文               例：这个医学 NER 任务的几万条样本
        数据量大 (GB~TB)                  数据量小 (MB 级)
        成本高，领域越深越值                成本低，几乎总是有效
                 │                                  │
                 └─────────────┬────────────────────┘
                               ▼
                    下游任务微调 (带 label)
                               │
                               ▼
                       更高的 task accuracy

最佳实践：DAPT → TAPT → 微调（三者叠加）
最省实践：TAPT → 微调（数据只需任务训练集，人人可做）
```

### 一句话区分

- **DAPT (Domain-Adaptive Pre-Training)**：用目标**领域**的全部数据做预训练。数据大、成本高、收益大。适合你真的要深耕一个领域（医疗/法律/金融这种），愿意攒一份领域语料。
- **TAPT (Task-Adaptive Pre-Training)**：只用**目标任务**的训练集数据做预训练。数据小、几乎免费、效果稳定。适合数据少、或想低成本榨干基座。
- 还有个增强版 TAPT：任务训练集太小 → 用数据增强扩充后再 TAPT（论文里叫 *TAPT on augmented data*）。

## 关键实验设计

```
领域 (4 个，刻意选了"和通用预训练分布差很远"的):
  - Biomedical 论文 (BIOMEDICA)
  - CS 论文 (S2ORC)
  - News (新闻)
  - Reviews (评论，如 Amazon/Yelp)

每个领域下挂多个分类任务 (文本分类为主):
  - 医疗: ChemProt, RCT, ...
  - CS: ACL-ARC, SCIERC, ...
  - 新闻: Hyperpartisan, AGNews, ...
  - 评论: Amazon reviews, IMDB, ...

对比维度:
  baseline (只微调)  vs  +TAPT  vs  +DAPT  vs  +DAPT+TAPT
```

实验刻意跨 4 个差异很大的领域，是为了证明这个结论**不是某个领域的偶然**，而是普遍规律。

## 关键发现（我提炼的 4 条）

### 1. "接着练"几乎总是涨点

即使 RoBERTa 已经是当时最强基座之一，在领域/任务数据上再 MLM 一遍，绝大多数任务都有提升。**这是反直觉的**——直觉上"基座已经见过这么多数据了，再练这点领域数据能有多大用？"答案是：有用。

### 2. TAPT 性价比最高

TAPT 只用任务自己的训练集（几万条样本），成本极低，却能稳定涨点。**如果你只能做一件事，做 TAPT。**

### 3. DAPT + TAPT > 两者单独

两者不是互斥，而是互补：DAPT 给领域全局视野，TAPT 给任务聚焦。叠加效果最好。

### 4. 收益取决于"领域距离"

```
领域与通用预训练分布的"距离"越大 → CPT 收益越大

  通用评论 (Amazon)   ──距离近──>  收益小 (本来就在分布内)
  生物医学论文         ──距离远──>  收益大 (大量专业术语/句式)

启示: 越专业、越垂直、行话越多的领域，CPT 越值。
      通用任务/常见领域，CPT 的边际收益就小。
```

## 为什么有效？（直觉）

```
通用预训练: 模型见过"人话"，但没怎么见过"医生/律师/金融分析师的话"
           → 词表里有这些词，但分布没对齐

CPT (继续 MLM):
  - 让模型适应领域的词汇分布、句式、概念组合
  - 不是"学新知识"，而是"把领域语言对齐到模型已有的表示空间"
  - 下游微调时，梯度不用再花成本去"适应领域"，直接学任务

类比: 你英语很好（通用预训练），但刚进医学院，
      先读一年医学文献（CPT），再上临床课（SFT）——
      不是为了学英语，是为了让英语"在医学语境下流畅"。
```

## 什么时候 CPT 有用 / 没用

| 场景 | CPT 是否值得 |
|------|-------------|
| 领域专业、行话多（医疗/法律/金融/代码） | ✅ 强烈值得 |
| 任务数据极少（几千条） | ✅ TAPT 几乎免费涨点 |
| 领域和通用分布接近（普通评论、新闻摘要） | ⚠️ 边际收益小 |
| 任务本身不需要领域知识（通用 QA、情感分类） | ❌ 收益有限 |
| 基座已经在该领域很强（如医学专用基座） | ⚠️ DAPT 收益下降，TAPT 仍可做 |

## 方法细节（怎么训）

```
1. 拿一个已经预训练好的 LM (RoBERTa)
2. 在领域/任务数据上继续 MLM (Masked Language Modeling) 目标
   - 注意：这一步不用任何 task label，纯自监督
3. 学习率：比原始预训练小（重要！见 week9_cpt_concepts.md）
4. 训完保存 checkpoint
5. 在这个 checkpoint 上做下游任务的标准微调（带 label）
```

**关键点**：CPT 用的是 **MLM 自监督**，不碰任务标签。任务标签留到最后的微调阶段。这是 CPT 和 SFT 的本质区别——CPT "学语言"，SFT "学任务"。

---

## 实战要点（我以后做 CPT 会怎么用）

1. **先 TAPT，再考虑 DAPT**。TAPT 成本极低（就用任务训练集），先把这个免费涨点拿到手。
2. **领域越垂直，越值得攒 DAPT 语料**。医疗/法律/金融这种，领域语料的 ROI 极高。
3. **CPT 学习率要小**。别用预训练的大 lr，会冲坏基座（详见 concepts 笔记）。
4. **CPT 是无标签的**。手里的领域文本即使没标注，也能用来 CPT——这是它最大的杠杆：把"无标注领域文本"变成"涨点"。

---

## 我的 takeaway

1. **"预训练不是终点"** —— 这篇论文的标题就是结论。通用基座再强，到专业领域都得"接着练"。
2. **TAPT 是被严重低估的免费午餐**。只用任务训练集、成本极低、几乎稳定涨点。我以后做任何领域任务，TAPT 应该是 baseline 之一。
3. **领域距离决定 ROI**。判断要不要做 CPT，先问"我的领域和通用分布差多远"。差越远，越值。
4. **为 BloombergGPT 和 HuatuoGPT-II 提供了理论基础**。这篇是 2020 年（encoder-only 时代），后两篇把它扩展到了 decoder LLM + 生成任务。三篇串起来读才完整（见 `week9_cpt_concepts.md`）。
