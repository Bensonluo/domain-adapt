## 一、CPT 在训练管线中的位置

```
┌──────────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  通用预训练    │ -> │   CPT   │ -> │   SFT   │ -> │  对齐    │
│ (海量通用语料) │    │(领域语言)│    │(任务指令)│    │(DPO/GRPO)│
└──────────────┘    └─────────┘    └─────────┘    └─────────┘
   知道"人话"      理解"领域话"    会"做任务"      做得"合偏好"

CPT 的角色: 把模型从「会说通用语言」变成「理解领域语言」。
            它发生在 SFT 之前，用「无标签领域文本」做自监督。
            —— 不学任务、不学偏好，只补"领域语感"。
```

**三篇论文分别覆盖 CPT 的不同切面**：

| 论文 | 时代 | 切面 | 一句话 |
|------|------|------|--------|
| Don't Stop Pretraining (2020) | encoder-only | **要不要做 CPT** | 几乎总是值得，TAPT 最便宜 |
| HuatuoGPT-II (2023) | decoder LLM | **CPT 数据格式** | 格式要对齐下游（ChatML），否则知识取不出 |
| BloombergGPT (2023) | decoder LLM | **CPT 数据配比** | 保住通用（~1:1），别纯领域 |

## 二、灾难性遗忘 (Catastrophic Forgetting)

### 成因

```
CPT 时，模型参数被领域数据"拉着走"，原来预训练学到的通用表示被覆盖。

  预训练后:  参数 = 通用知识的"稳态"
  CPT 时:    领域梯度持续推动参数 → 偏离通用稳态 → 通用能力下降

  机制: 神经网络是"参数共享"的，领域和通用用同一批参数。
        领域数据训练多了，参数就向"领域最优"漂移，通用表现必然受影响。
```

### 量化

```
遗忘率 = (通用能力_before - 通用能力_after) / 通用能力_before

例:
  MMLU(CPT前) = 60%   MMLU(CPT后) = 54%
  → 遗忘率 = (60-54)/60 = 10%

  领域 benchmark: 40% → 65%  (领域涨 25 个点)
  ──────────────────────────────────────────
  净收益: 领域 +25, 通用 -6   → 整体值得，但要监控遗忘

  什么时候不值: 领域只涨 5 点、通用掉 10 点 → CPT 是负收益，停下来。
```

### 4 种缓解方法（按实用性排序）

```
┌───┬──────────────────────┬───────────────────────────────────────┬──────────┐
│   │ 方法                  │ 机制                                   │ 实用度    │
├───┼──────────────────────┼───────────────────────────────────────┼──────────┤
│ 1 │ Replay (混入通用数据)  │ CPT 时掺 30-50% 通用语料，让模型        │ ⭐⭐⭐⭐⭐ │
│   │                      │ 「边学领域边复习通用」                   │ (最实用)  │
│   │                      │ —— BloombergGPT 把它发挥到极致(1:1)     │          │
├───┼──────────────────────┼───────────────────────────────────────┼──────────┤
│ 2 │ 调小学习率            │ 小 lr = 参数漂移幅度小 = 通用破坏少      │ ⭐⭐⭐⭐  │
│   │                      │ 预训练 lr 的 1/10 ~ 1/100              │ (必做)   │
├───┼──────────────────────┼───────────────────────────────────────┼──────────┤
│ 3 │ LoRA-based CPT        │ 冻结基座，只训少量 LoRA 参数             │ ⭐⭐⭐⭐  │
│   │                      │ 通用知识「物理上」不动(冻结)，天然防遗忘  │ (主路线)  │
├───┼──────────────────────┼───────────────────────────────────────┼──────────┤
│ 4 │ EWC (正则)            │ 对"重要参数"加正则，惩罚其大幅变动        │ ⭐⭐      │
│   │                      │ 理论优雅，工程上被前 3 个盖过            │ (了解)   │
└───┴──────────────────────┴───────────────────────────────────────┴──────────┘
```

**我的默认组合**：Replay（混通用）+ 小 lr + LoRA。三者叠加，遗忘基本可控。EWC 了解即可，实战少用。

## 三、数据混合比例 vs 遗忘率（自测题 3 的答案）

```
预期 trade-off 曲线 (领域提升 vs 通用遗忘):

  领域提升 ───────────────╱──────── 领域 benchmark
                         ╱         (↑ 越多领域数据越涨，边际递减)
                        ╱
                       ╱
                      ╱
  ───────────────────╱───────────────────────────── 遗忘率(右轴↓)
                    ╱     ╲
                   ╱       ╲  通用遗忘
                  ╱         ╲(↑ 领域占比越高，通用掉越多)
                 ╱           ╲
                ╱             ╲
   ──────┬─────┬──────┬───────┬───────── 领域占比
        0%    30%    50%     70%   100%
                    ↑
                 BloombergGPT ~49% 落点(均衡)

读图:
  - 纯领域 (100%): 领域最高，但遗忘也最大 → 高风险
  - 70-30 (领域-通用): 领域够强、遗忘可控 → 实战常用均衡点
  - 50-50: 遗忘最低，但领域提升也小 → 保守/数据量极大时
  - 0% (不 CPT): 无遗忘也无领域提升 → 基线

核心: 这是反比关系，没有免费午餐。
      选点 = 在「领域增益」和「通用遗忘」之间找你的应用能容忍的平衡。
      BloombergGPT 选 ~1:1，是因为它「两者都要」。
```

**经验值**（综合三篇 + 后续研究）：

| 配比 | 领域提升 | 遗忘风险 | 适用 |
|------|---------|---------|------|
| 纯领域 100% | 最大 | 最高 | 数据极少、只在乎领域、不要通用 |
| 70-30 | 大 | 可控 | **实战默认**，性价比最高 |
| 50-50 | 中 | 低 | 数据量大、且通用也要保（BloombergGPT） |

## 四、为什么 CPT 学习率要比预训练小 10-100 倍（自测题 1）

```
关键区别: 「在什么状态上学」

  预训练:  参数是随机初始化的「白纸」
           → 离最优解很远 → 需要大步走（大 lr）才能快速逼近
           → 大 lr 不会"破坏"什么，因为本来就没有"学好的东西"可破坏

  CPT:    参数已经是预训练训好的「成品」(通用知识已编码)
           → 已经在一个好的解附近
           → 大 lr = 大步走 = 直接把成品踩碎 = 灾难性遗忘
           → 小 lr = 小步微调 = 在不破坏通用表示的前提下，吸收领域分布

类比:
  预训练 = 在空地上盖楼，可以大力施工
  CPT    = 在已盖好的楼上做装修，必须轻手轻脚，否则砸坏承重墙(通用知识)

数量级:
  预训练 lr: ~1e-3 ~ 1e-4 量级
  CPT lr:   ~1e-5 ~ 1e-6 量级  (小 10-100 倍)
  SFT lr:   通常比 CPT 再大一点(因为带任务信号,可更激进)

如果用预训练 lr 做 CPT 会怎样?
  → loss 一开始降得快（领域学到了）
  → 但通用 benchmark 急剧下降（遗忘严重）
  → 模型"领域变强、变蠢" → 净收益可能为负
```

## 五、TAPT vs DAPT（自测题 2）

```
                预训练基座
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
     TAPT                     DAPT
  「任务训练集」继续 MLM     「整个领域」继续 MLM
   数据量: 小 (MB)            数据量: 大 (GB~TB)
   成本:   极低                成本:   高(需攒领域语料)
   收益:   稳定小涨             收益:   大(领域越深越大)
   精度:   与任务强相关         精度:   领域全局视野
        │                       │
        └───────────┬───────────┘
                    ▼
            DAPT → TAPT → 微调  (最强组合)
```

**适用场景**：
- **TAPT**：数据少、想低成本榨干基座、或作为任何领域任务的默认 baseline。
- **DAPT**：数据充足、要长期深耕一个领域、愿意投入攒领域语料。

## 六、CPT 决策框架（我自己的 checklist）

做任何领域任务前，先回答这 5 个问题：

```
Q1: 我的领域和通用分布「差多远」？
    差远(医疗/法律/金融/代码) → CPT 价值大，继续
    接近(普通评论/新闻)       → 先做 TAPT 即可，不必 DAPT

Q2: 我有多少「无标注领域文本」？
    < 100MB   → 只够 TAPT，别想 DAPT
    100MB~10GB → TAPT + 轻量 DAPT
    > 10GB    → 正经 DAPT，值得投入

Q3: 数据「格式」对齐下游了吗？(HuatuoGPT-II 启示)
    下游是对话/助手 → CPT 数据用 ChatML/instruction 格式，别用纯文本续写
    下游是理解/分类 → 纯文本 CPT 也 OK

Q4: 通用数据「replay」比例多少？(BloombergGPT 启示)
    默认 70(领域)-30(通用) 起步
    若通用也要保 → 调到 50-50
    绝不纯领域(100%)，除非完全不要通用

Q5: 怎么「防遗忘」？
    默认三件套: Replay + 小 lr(预训练的 1/10~1/100) + LoRA
    并在 week12 同时评领域 benchmark + 通用 benchmark(MMLU) 量化遗忘
```

## 八、三篇论文的一句话串讲

```
Don't Stop Pretraining (2020)
  → 「要不要做 CPT」: 要。TAPT 便宜且几乎总涨点。领域越垂直越值。

HuatuoGPT-II (2023)
  → 「CPT 数据怎么准备」: 格式要对齐下游(对话)，否则知识学进去取不出。
     无标注文本用 LLM 转成 instruction，榨干领域资产。

BloombergGPT (2023)
  → 「CPT 数据怎么配比」: 保住通用(~1:1)，别纯领域。质量>数量。双线评估。

三者合起来 = 一套完整 CPT 方法论:
  做不做(DSP) → 怎么造数据(HuatuoGPT) → 怎么配比(Bloomberg)
```
