# BloombergGPT 论文精读笔记

> 论文: BloombergGPT: A Large Language Model for Finance (Wu et al., 2023)
> 链接: https://arxiv.org/abs/2303.17564
> 机构: Bloomberg

---

## 核心观点 (TL;DR)

```
BloombergGPT 严格说不是 CPT，是「从零训练」一个 50B 金融大模型。
真正的创新不在模型架构，而在「数据策略」:

   混合数据 = ~51% 通用语料 + ~49% 金融专有语料 (FinPile)
              通用是地基，金融是专长，刻意保持接近 1:1

关键结论: 金融任务 SOTA，通用任务不掉。
   —— 正面回答了「灾难性遗忘」: 与其 CPT 后补救遗忘，
      不如从一开始就把通用数据混进去，从源头避免。

一句话: 领域能力的天花板，由你愿不愿意保住通用能力决定。
       砍通用去堆领域 = 短期领域涨、长期天花板低。
```

---

## 核心问题

做金融 LLM，有两条路：
- **路线 A（CPT）**：拿个通用 LLaMA/GPT，在金融数据上继续预训练。便宜，但有遗忘风险。
- **路线 B（从零）**：自己从头训一个，数据和配方完全可控。贵，但能从根本上决定模型"长什么样"。

Bloomberg 选了 B。问题变成：**从零训，通用和金融数据该放多少？纯堆金融行不行？**

---

## 为什么选"从零训练"而不是 CPT

```
Bloomberg 的特殊优势:
  - 有几十年积累的金融专有数据 (Bloomberg 终端、新闻、研报、SEC 文件...)
    —— 这是别人没有的护城河，值得为之专门训一个模型
  - 数据量足够大 (FinPile ~363B tokens)，撑得起从零训练
  - 算力 / 工程能力到位 (大厂)

从零 vs CPT 的权衡:
                  从零训练              CPT
  数据控制        完全可控 ✅           受限于基座预训练已有分布
  通用-领域平衡   从一开始就设计好 ✅   事后补救，容易遗忘 ⚠️
  成本            极高 ❌              低 ✅
  适用场景        有海量专有数据+算力    数据/算力有限，或只想增量提升

启示: 对个人/小团队，CPT 仍是现实选择（我们 Phase1 走 CPT 路线）。
      BloombergGPT 是"理想态"，但它的数据策略对 CPT 同样有指导意义 ——
      即使做 CPT，也该混入通用数据（replay），理念是一致的。
```

## 模型 & 数据规模

```
模型:
  - 参数量: ~50B (50 billion)
  - 架构: 基于 BLOOM 的 decoder-only Transformer
  - 定位: 当时金融领域最大的从零训练 LLM

数据 (这是本文真正的核心):
  - 总训练 token: ~700B
  - FinPile (金融专有): ~363B tokens, 占 ~49% (论文口径约 48.7% 金融 / 51.3% 通用，
    因去重/统计口径略有出入，核心是「接近 1:1，通用略多」)
  - 通用语料: ~345B tokens, 占 ~51%
```

### FinPile 的构成（金融数据来源）

```
Bloomberg 几十年积累的金融数据，覆盖:
  - Bloomberg News (新闻)
  - SEC Filings (美国证监会披露文件: 10-K, 10-Q...)
  - Press Releases (公司新闻稿)
  - Web-crawled financial content (网页金融内容)
  - Bloomberg 终端的其他专有金融文本

特点: 多源、专业、覆盖广，且经 Bloomberg 自有清洗管线处理。
       —— 这是别人复刻不了的资产，也是"从零训练值得"的根本原因。
```

### 通用语料的构成

```
并非自建，而是汇集公开数据集:
  - The Pile
  - C4
  - RefinedWeb
  - Wikipedia
  - 其他公开语料

为什么保留这么多通用? 见下一节 —— 这是全文最关键的设计决策。
```

## 混合数据策略（全文灵魂）

```
❌ 错误直觉:
   "做金融模型 → 应该 90%+ 金融数据 → 最大化领域能力"

✅ Bloomberg 的反直觉决策:
   通用 ~51% + 金融 ~49%  (通用还略多!)

为什么?
  ┌─────────────────────────────────────────────────┐
  │  通用能力 = 领域能力的「地基」                       │
  │                                                   │
  │  金融任务不只需要金融知识，还需要:                  │
  │   - 推理 / 常识 / 语言理解 (来自通用语料)           │
  │   - 指令遵循 / 对话能力 (来自通用语料)              │
  │   - 跨领域的泛化 (来自通用语料)                     │
  │                                                   │
  │  砍通用去堆金融 → 地基塌了 → 金融天花板反而低       │
  └─────────────────────────────────────────────────┘

这就是为什么 BloombergGPT 金融 SOTA 的同时，
通用 benchmark (MMLU, BIG-bench Hard 等) 也不输同规模通用模型。
```

### 这对"灾难性遗忘"的启示

```
灾难性遗忘的传统解法 (CPT 场景):
  1. 学习率调小
  2. Replay buffer: 混入 10-30% 通用数据
  3. EWC 正则
  4. LoRA (只动少量参数)

BloombergGPT 的"终极解法":
  因为是从零训，根本不存在"先学了通用再被金融覆盖"的问题 ——
  通用和金融从一开始就交织在一起 (shuffled mixed training)。
  模型从来没"只学通用"过，所以也没有"遗忘通用"这一步。

启示给 CPT: 即便走 CPT 路线，也要混入足量通用数据 (replay)，
          而不是纯领域 CPT。比例可参考 Bloomberg 的 ~1:1。
```

## 数据质量流程（工程细节）

```
金融数据对"正确性"极敏感 (一个数字错 = 法律/财务风险)，所以清洗很重:

1. 去重 (deduplication): 文档级 + 段落级，去掉重复内容
2. 过滤低质量: 规则 + 启发式过滤乱码/过短/模板化内容
3. 去除 PII / 敏感信息
4. 语言识别 / 分语种处理
5. 配比与 shuffle: 通用和金融混合后打乱，避免训练中先吃完一类再吃另一类

关键理念: 「数据质量 > 数据数量」。与其堆更多噪声数据，不如把已有的洗干净。
        —— 这条对个人做 CPT 同样适用: 先清洗，再谈量。
```

## 评估体系（FLUE）

```
Bloomberg 提出 FLUE (Financial Language Understanding Evaluation):
  = 金融领域专用 benchmark 套件

包含任务 (覆盖金融 NLU + NLG):
  - 情感分析 (Sentiment): FiQA, FPB (Financial PhraseBank)
  - 新闻标题分类 (Headline)
  - 金融 QA / 推理: ConvFinQA (对话式金融数值推理)
  - 命名实体识别 (NER)
  - 其他金融理解任务

同时还在通用 benchmark 上评估:
  - MMLU, BIG-bench Hard, Reading Comprehension, Translation, ...

双线评估的目的:
  证明「金融强」的同时「通用没掉」 —— 这才是混合数据策略成功的证据。
  只评金融不评通用 = 无法证明你没牺牲通用能力。
```

### 关键结论（结果）

```
金融任务 (FLUE): SOTA，明显超过同规模通用模型 + 同期金融模型
通用任务 (MMLU 等): 与同规模通用模型持平，部分任务甚至更好

→ 混合数据策略验证成功:
   "鱼和熊掌兼得" —— 领域强 + 通用不掉
   核心证据 = 不是靠牺牲通用换来的领域能力。
```

## 实战启示（我做 CPT 会怎么用）

1. **永远保留通用数据**。做金融/医疗/法律 CPT 时，别用 100% 领域数据。混入 30-50% 通用（replay）。这是本篇给我最直接的配方。
2. **数据质量 > 数量**。先花力气清洗、去重、过滤，再谈加量。脏数据越多越伤。
3. **双线评估**。评估领域模型时，一定要同时评通用 benchmark。只看领域指标 = 自欺欺人，可能通用已经塌了。
4. **混合 = shuffled**。通用和领域数据要在训练中充分打乱，不要"先通用后领域"（后者就是制造遗忘）。
5. **现实定位**：我没有 Bloomberg 的算力和专有数据，走不了"从零"。但它的**比例直觉**（~1:1）和**质量理念**直接可用于我的 CPT（week10-12）。

---

## 我的 takeaway

1. **"砍通用堆领域"是新手陷阱**。BloombergGPT 用 ~1:1 的比例证明：保住通用才是领域天花板的关键。这纠正了我"做领域就该 90% 领域数据"的直觉。
2. **从零 vs CPT 是 cost 的选择，不是理念的选择**。理念（混合 + 保通用）两边通用。我没钱从零，但完全可以在 CPT 里 replay 通用数据，吃到同样的红利。
3. **数据是真正的护城河**。BloombergGPT 的 50B 模型别人能仿，但 FinPile（几十年金融专有数据）仿不了。这提醒我：做 domain adaptation，**花在攒/清洗领域数据上的时间，回报远高于调超参**。
4. **双线评估是标配**。我以后任何 CPT 实验，都要同时报领域 benchmark + 通用 benchmark（MMLU）—— 这正是 week12「灾难性遗忘量化」要做的事，本篇给了理论依据。
